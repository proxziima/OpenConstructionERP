# Confirmed PostgreSQL bugs — ALL RESOLVED (2026-05-30)

All 15 confirmed dialect-divergence bugs below are fixed. Summary of fixes:

* **Tolerant numeric cast** — new `app/core/sql_numeric.py::numeric_value()` replaces
  `cast(text, Float)` in `catalog/repository.py`, `costs/repository.py` (rate range)
  and `projects/router.py` (budget/PO/BOQ rollups). On PG it guards with a regex so a
  non-numeric string becomes `0` instead of raising `invalid input syntax`.
* **Aware-datetime coercion** — new `app/core/db_types.py::AwareDateTime` coerces ISO
  strings / naive datetimes to tz-aware at bind time; applied to every
  `daily_diary` timestamp column and `fieldreports.approved_at`.
* **audit `_parse_iso`** now returns tz-aware datetimes (assumes UTC when offset-less).
* **costs `category_tree`** groups by the JSONB extraction *expressions*, not the
  SELECT aliases.
* **costs `search_for_autocomplete`** uses `jsonb_array_length` on PG (dialect detected
  from the *session* bind, not the global engine) — `json_array_length(jsonb)` does not
  exist on PG.
* **project_intelligence `_collect_documents`** uses `string_agg` on PG (no GROUP_CONCAT),
  dialect detected from the session bind.
* **qms** writes normalise every timestamp to UTC-with-offset (`_to_utc_iso`) so the
  String(32) columns sort chronologically under lexical comparison on both dialects.
* **bim_hub** dynamic-group filtering dropped the PG `@>` fast-path (it was case- and
  type-strict, diverging from the case-insensitive/wildcard Python predicate) — both
  dialects now share the bounded Python filter.
* **seed_demo_v2 `_bulk_insert_links`** runs against the configured DB via SQLAlchemy
  core `ON CONFLICT DO NOTHING` instead of a hardcoded SQLite file.

Regression coverage: `backend/tests/pg/test_pg_bug_fixes.py` (PG lane, 7 new tests, all
green); existing SQLite module suites still pass.
