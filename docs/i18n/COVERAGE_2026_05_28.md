# i18n Coverage Audit — 2026-05-28

EN source has **7557** keys (file: `frontend/src/app/locales/en.ts`).

## Summary of this pass

A focused translation pass shipped on 2026-05-28 covered the highest-impact
UI surfaces — the ones rendered on every screen of the app:

- `nav.*` — phase labels (Estimation/Planning/Execution/Closure), group
  labels (Communication / Documents / Commercial / Property / Operations /
  Analytics) and their sub-headlines.
- `common.*` — Cancel / Save / Delete / Edit / Confirm / Close / Loading /
  Search / Error / Success / Actions / Add / Description / Back / Continue /
  Copied / Create / Duplicate / Export / Filter / Import / Item /
  Archive / Connection error / Unknown error / How it works.
- `sidebar.admin_grid.*` — Users / Audit / Roles / Rules / Modules /
  Integrations / Settings / About + Edit menu / Save / Cancel / Show
  hidden / Geo Hub.
- `login.brand_*` — Customize logo / White-label branding / Upload logo /
  Company name / Reset to default / size + type validation errors,
  GitHub source code, Community help & discussion.
- `tour.*` — Next / Previous / Finish / Skip (the on-boarding tour).
- `whatsnew.*` — Title / Dismiss.

Applied across **all 26 non-EN locales**. One commit per locale. Each
diff is small but covers the surfaces the user encounters first.

### Limitations / pending work

1. The biggest backlog is **missing keys** — e.g. `ro/hr/id/th/vi/da/no/
   cs/sv/fi/bg/tr/hi/mn` are missing ~1,554 keys each (the keys never got
   added to those locale files in earlier waves). The task constraints
   forbade *adding* new keys, so these gaps remain. They need a separate
   one-shot pass that copies the EN baseline into each locale file before
   translation.
2. `boq.*` (~14,000 untranslated occurrences across all locales),
   `propdev.*` (~8,600), `match_elements.*` (~4,200), `accommodation.*`
   (~3,900), `costs.*` (~3,200) are domain-specific surfaces. These need
   careful native review and are too risky to bulk-translate without
   subject-matter input — left for a follow-up wave.
3. Country-specific exchange labels (`nav.au_boq_exchange`,
   `nav.de_din`, `nav.fr_dpgf_exchange`, `nav.uk_nrm_exchange`, etc.)
   were **intentionally left as English** — the country code + format
   name (e.g. "GAEB Exchange", "NRM 3420 Exchange") is the recognized
   industry identifier and translating "Exchange" to "Austausch" /
   "Échange" would harm recognizability for cross-border users.
4. `nav.mode_pro_badge` ("PRO") and `nav.mode_std_badge` ("STD") were
   **intentionally left untranslated** — they are compact 3-letter pill
   badges and non-Latin translations would break the layout.
5. Japanese (ja) — the well-known #245 backlog of ~1,700 untranslated keys
   was reduced by ~6 in this pass (common.* gains). The bulk of ja keys
   are domain-specific and need a dedicated native pass.
6. `tour.*` keys reported "102 untranslated" in some locales actually
   means they are *missing*, not EN-equal. Cannot fill without adding
   new keys.
